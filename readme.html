Atomic Widget

Atomic Widget plugin gives every widget an extra control field called "Atomic Widget" that lets you control the pages that the widget will appear on. The text field lets you use Hybrid Core "Atomic" Context by Justin Tadlock.

How it works

For each widget you can add string of contexts separated by comma ,.

To exclude a context you can use ! in front of the target context

The context you add will override the previous context. For example if you want to display widget in all category archive but not in "uncategorized" category, you can use taxonomy-category, !taxonomy-category-uncategorized

If you want to display the widget in all pages, you can use wp or simply left it blank.

wp context is useful if you need to disable widget in one or several pages. For example if you want to display widget in all pages except in category "uncategorized" archive, you can use wp, !taxonomy-category-uncategorized

Other Example

Blog page: blog

Front page: home

Archive pages: archive

Category archive: taxonomy-category

Category archive with slug "x": taxonomy-category-x

Multiple category archive with slug "x" or "y": taxonomy-category-x,taxonomy-category-y

Singular post: singular-post

Exclude singular pages: wp,!singular

Exclude Home Page: wp,!home

All singular except page (post type): singular,!singular-page

You can view list available context in Theme Hybrid Documentation ( member only )

Available Filter

  1. atomic_widget_display_{widget-id}
    filter to modify output for each widget (bool, true/false)
  2. atomic_widget_{widget-id}
    filter to modify "atomic_context" instance input for each widgets (string)
  3. atomic_widget_context
    filter to modify contexts without adding it to hybrid core contexts. (array)
  4. atomic_widget_conditional
    filter to modify conditional output (bool, true/false)
  5. atomic_widgets (require PHP 5.3)
    bulk filter to modify each widgets in one array.
  6. atomic_widget_disable_sidebar
    filter to disable sidebar if the sidebar have no widget. default: "true" (bool,true/false)

You can view list available context in Theme Hybrid Documentation ( member only )
